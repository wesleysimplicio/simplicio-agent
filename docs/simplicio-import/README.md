this repo, per the fusion rule in epic #18: no blind copy — every novelty
gets an issue before implementation, and every wave gets a dated log here.

  updated as new waves land or new candidates are found).
- `YYYY-MM-DD-turbo-import-log.md` — one file per audit wave: what was
  analyzed, what was imported, what was rejected, and the evidence.
