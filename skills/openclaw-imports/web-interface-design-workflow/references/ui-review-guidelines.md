# Web design review notes

Use when auditing existing UI for guideline compliance, accessibility, UX quality, or design consistency.

## Core rules
- inspect actual files when possible
- fetch fresh guideline sources when the workflow depends on an external standard
- report actionable findings, not vague taste opinions
- include accessibility and usability, not only visual polish

## Good review flow
1. fetch/confirm the governing guideline
2. inspect target files
3. record findings with explicit rationale
4. suggest concrete remediations
