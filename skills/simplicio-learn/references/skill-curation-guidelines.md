# Skill curation guidelines

Use this as the compact reference when deciding whether a lesson belongs in a skill, a support file, or memory.

## Shape rule
- **Class-level skill**: recurring workflow or task class; keep the SKILL.md durable and reusable.
- **references/**: session-specific detail, reproduction notes, concise knowledge banks, external excerpts, and examples that would bloat the main skill.
- **templates/**: starter artifacts to copy and adapt.
- **scripts/**: deterministic probes, verifiers, and generators.

## Update rule
1. If a skill was loaded this session and it was missing a step or pitfall, patch that skill first.
2. If the lesson applies to a broader workflow, widen the umbrella skill instead of creating a one-session skill.
3. If the detail is useful but too specific for the main skill, move it into references/.
4. If multiple skills overlap, keep one umbrella and add cross-links instead of proliferating narrow entries.

## Example from this session
- Imported Asolaria patterns were folded into the existing learning/curation umbrella instead of being left as narrow one-off skills.
- The durable form is: evidence-gated halting, agent-orchestration-as-data, and end-of-session consolidation with durable memory.
