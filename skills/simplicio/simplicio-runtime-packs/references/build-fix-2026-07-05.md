## Erros e Correções

### 1. memory_v2.rs — 7 erros

**Problema 1: store_embedding cortada ao meio**
A função `pub fn store_embedding(` na linha 397 abria sem fechar. O corpo dela
(`&self, item_id, chunk_id, model, embedding...`) estava ÓRFÃO depois de `search_vector`.
`search_vector` foi inserida DENTRO de `store_embedding`.

**Correção:** Reunificar corpo + cabeçalho. Usar `simplicio edit` com replace
no bloco completo:

```json
{"op":"replace","find":"pub fn store_embedding(\n\n    // ------...","with":"pub fn store_embedding(\n        &self,\n        item_id: i64,..."}
```

**Problema 2: "search-parallel" braço de match perdido**
O braço `"search-parallel" => {...}` aparecia dentro de `println!()` do braço
`"search"` — não era um braço de match válido. O `println!(` abria mas nunca
fechava, e o conteúdo entre eles foi perdido.

**Correção:** Reescrever ambos os blocos:

```json
{"op":"replace","find":"if json {\n                println!(\n\n        \"search-parallel\" => {","with":"if json {\n                println!(\"{}\", serde_json::to_string_pretty(&results).map_err(|e| e.to_string())?);\n            }\n        }\n        \"search-parallel\" => {"}
```

**Problema 3: db_path moved (use after move)**
`let db_path = self.db_path.clone()` (linha 288). Usado na primeira closure
`move ||` (linha 315). Na segunda closure (linha 346), `db_path` já foi movido.

**Correção:** Clonar antes de cada closure `move`:

```rust
let db_path_fts = db_path.clone();  // antes da 1ª closure
let fts_handle = std::thread::spawn(move || {
    let conn = Connection::open(&db_path_fts).ok()?;
    // ...
});
let db_path2 = db_path.clone();  // antes da 2ª closure
let vec_handle = std::thread::spawn(move || {
    let conn = Connection::open(&db_path2).ok()?;
    // ...
});
```

### 2. action_gate.rs — 2 erros

**Problema:** Struct `ActionGateEvaluation` tem campos `&'static str`:
```rust
pub(crate) struct ActionGateEvaluation {
    pub(crate) risk_class: &'static str,
    pub(crate) decision: &'static str,
    // ...
}
```
Mas `cached_result.risk_class` e `cached_result.decision` são `String` do
`CachedDecision` em `src/decision_cache.rs`. `&cached_result.risk_class` não
é `'static` — compilador rejeita com E0597.

**Solução — Box::leak:**
```rust
let risk_class: &'static str = Box::leak(cached_result.risk_class.clone().into_boxed_str());
let decision: &'static str = Box::leak(cached_result.decision.clone().into_boxed_str());
```

**⚠️ Trade-off:** Memory leak intencional. Aceitável para strings pequenas (<1KB, <100).

**Não tentar:** Mudar a struct para `String` — isso quebra 10+ match arms
em `src/commands/run.rs`, `src/delivery/dod.rs`, `src/wave_publish.rs`, etc.

### 3. commands/mod.rs — 1 erro

**Problema:** `dispatch()` move `args` dentro do `match` (primeira branch
chama `resolve_config(args)` que toma ownership). Depois, código de fallback
tenta `args.clone()` — E0382 (use after move).

**Correção:** Clonar ANTES do match:
```rust
let saved_args = args.clone();  // mover para ANTES do match
let result = match command {
    "doctor" => {
        let config = resolve_config(args)?;  // args moved aqui
        // ...
    }
    // ...
};
// Usar saved_args no fallback:
let args_clone = saved_args.clone();  // em vez de args.clone()
```

### 4. chunk_15.rs — 1 erro

**Problema:** `capabilities_cache().lock()` retorna `MutexGuard` sem `mut`,
mas `cache.insert()` precisa de mutabilidade.

**Correção:**

```rust
// Antes:
if let Ok(cache) = capabilities_cache().lock() {
// Depois:
if let Ok(mut cache) = capabilities_cache().lock() {
```
